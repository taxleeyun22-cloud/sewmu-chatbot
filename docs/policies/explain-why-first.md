# 💡 "왜" 부터 설명 룰 (Auto Mode 핵심)

**시행**: 2026-05-09 (사장님 결정)

## 룰

새 도구 / 패턴 / phase / 작업 도입 시 항상 **"왜"** 부터:

### 5단계 설명 순서 (절대)

1. **🚨 왜 필요한지** — 현재 어떤 문제 / 사고 시나리오
   - 구체적인 예시 + 실제 사고 가능성
   - 사장님 본업에 어떤 손해

2. **❌ 안 하면 어떻게 되는지** — 실제 영향 시나리오
   - "거래처가 9시에 챗봇 켰는데..." 같은 구체 시간선
   - 사장님 시간 손해 / 거래처 신뢰도 / 매출 영향

3. **✅ 도입 후 효과** — 비유 / 시나리오
   - 비전공자 사장님 친화 비유 (예: CCTV vs 직접 순찰)
   - Before / After 시간선 비교

4. **💰 비용 / 위험** — 솔직히
   - 무료? 유료? 시간? 학습 부담?
   - 위험은 정직하게 명시

5. **🔧 어떻게 하는지** — Step by Step
   - 1, 2, 3 순서
   - 비전공자도 따라 가능
   - 막힐 수 있는 곳 미리 안내

## 적용 예시 (참고)

```
"Sentry 가입하세요. → 1. https://sentry.io 접속 → 2. ..."
```

### 맞는 예
```
🚨 왜: 현재 거래처가 챗봇 에러 발생해도 사장님 모름. 자체 logger 는 20건 한도.
❌ 안 하면: 거래처가 카톡으로 보고 → 사장님 1시간 디버그.
✅ 효과: 에러 즉시 사장님 폰 알림. 거래처 신뢰도 ↑.
💰 비용: 무료 (월 5천 events).
🔧 진행: 1. https://sentry.io 접속 → 2. ...
```

## 예외

명백히 단순 작업 (예: "test 돌려봐", "build 해봐", "git push") 은 "왜" 생략 OK.
새 **도구 / 패턴 / 결정 / 인프라 / 룰** 도입 시는 무조건 "왜" 부터.
