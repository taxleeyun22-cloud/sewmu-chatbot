# 신고 검토표 시스템 명세

세무회계 이윤 사내 어플 — 종소세/법인세 신고 결재용 검토 시스템.

---

## 0. 배경

- 직원이 신고 작업 마치면 사장님(세무사)이 결재 검토.
- 작년 vs 올해 비교 + 누적 메모로 "작년에 어땠는지" 매년 자연스럽게 따라옴.
- 기존 앱 자산(Person/Business/ChatRoom/통합 메모/AI 요약) 그대로 활용.
- 새로 만드는 건 본질적으로 **"신고 Case 그릇을 채우는 일"** 하나.

---

## 1. 핵심 원칙

| 원칙 | 내용 |
|---|---|
| 신고 단위 | 종소세 = 사람(Person) 단위 / 법인세 = 업체(Business) 단위 |
| 한 사람 다업체 | 한 사람이 법인 + 개인사업체 N개 보유 가능. 종소세는 개인사업체들 자동 합산 |
| 메모 일관성 | 신고 메모는 기존 메모 시스템에 attach_to만 추가 (별도 메모 모델 안 만듦) |
| 첫 해 운영 | 작년 Case는 retro 입력 (PDF 업로드만 하면 자동 필드 채워지므로 부담 적음) |
| 출력 | A4 가로 1장 PDF — 결재용 검토표 |

## 2. 데이터 모델

### 2.1 Filing (신규 테이블/컬렉션)

```
Filing
├─ id
├─ type                   '종소세' | '법인세'
├─ 귀속연도               int (예: 2024)
├─ owner_type             'Person' | 'Business'
├─ owner_id               Person.id 또는 Business.id
├─ 포함사업체[]           Business.id 리스트 (종소세일 때만 사용)
├─ 자동필드 (JSON)
│   ├─ 수입금액 / 매출액
│   ├─ 소득금액 / 결산서당기순이익 / 각사업연도소득금액
│   ├─ 종합소득공제 / 과세표준
│   ├─ 산출세액
│   ├─ 적용공제감면[]      [{ 종류, 금액 }, ...]
│   ├─ 결정세액
│   ├─ 가산세 (있으면)
│   ├─ 기납부세액
│   ├─ 납부할세액
│   └─ 실효세율            계산값 (결정세액 / 수입금액)
├─ 결재상태               '작성중' | '결재대기' | '보관완료'
├─ 결재자_코멘트          text (선택입력)
├─ 작성자                 직원 id
├─ 결재자                 사장님 id (보관완료 시 기록)
├─ 결재일자               date (보관완료 시 입력)
├─ created_at
└─ updated_at
```

### 2.2 Memo (기존 모델 확장)

기존 메모에 다음 필드 추가:

```
Memo
├─ ... (기존 필드 유지)
├─ attached_to            { type, id }
│                         type: 'Person' | 'Business' | 'ChatRoom' | 'Filing'
├─ related_persons[]      Person.id 리스트 (통합뷰 인덱스)
├─ related_businesses[]   Business.id 리스트 (통합뷰 인덱스)
├─ related_chatrooms[]    ChatRoom.id 리스트 (통합뷰 인덱스)
└─ related_filings[]      Filing.id 리스트 (통합뷰 인덱스, 신규)
```

**카테고리 추가**: 기존(할일/거래처정보/전화/문서/이슈/약속/완료/상담)에 **'신고 코멘트'** 추가. Filing 메모는 이 카테고리 default.

### 2.3 인덱스 자동 채우기 규칙

메모 생성 시 백엔드/트리거에서 자동:

| 메모 attach 위치 | 자동 채워질 인덱스 |
|---|---|
| Person | related_persons = [본인] |
| Business | related_businesses = [본인], related_persons = [대표자Person] |
| ChatRoom | related_chatrooms = [본인], related_businesses = [연결업체들], related_persons = [주업체 대표자] |
| Filing | related_filings = [본인], related_persons/businesses = [owner + 포함사업체] |

→ 이러면 Person/Business/ChatRoom 페이지의 통합 메모는 한 쿼리(인덱스 매치)로 다 끌어옴.

## 3. 화면 동선

### 3.1 Person(사용자) 페이지 — 기존 페이지에 추가

**신고 Case 섹션** (현재 비어있음 - 채우기):
- 헤더: "🧾 신고 Case — 종소세"
- "+ 새 종소세 Case" 버튼
  - 모달 입력: 귀속연도 / 포함 사업체 다중선택 (이 사람 보유 사업체 자동 추천) / (옵션) 작년 Case 자동 참조
- Case 리스트 (귀속연도 내림차순)
  - 행: `[2024귀속] [결재완료] 수입 16.7억 · 결정세액 1,832만 · 실효세율 1.10%  →` 열기 버튼
  - 클릭 시 Filing Case 상세 페이지로 이동

기존 통합 메모 섹션은 그대로 유지. 단, **Filing 메모도 자동으로 통합 메모에 표시됨** (related_persons 인덱스 매치).

### 3.2 Business(업체) 페이지 — 기존 페이지에 추가

**신고 Case 섹션** (신규):

- 업체가 **법인사업자**면:
  - 헤더: "🧾 신고 Case — 법인세"
  - "+ 새 법인세 Case" 버튼 (귀속연도 / 작년 Case 자동 참조)
  - Case 리스트 (귀속연도 내림차순)

- 업체가 **개인사업자**면:
  - 안내 박스: "이 사업체는 대표자 [김대표]의 종소세에 포함되어 있습니다."
  - 링크: "→ [김대표] 종소세 Case 보기" (대표자 Person 페이지로 이동)

### 3.3 ChatRoom(상담방) 페이지 — 기존 페이지에 추가

**관련 신고 Case 섹션** (신규, 읽기 전용):

- 헤더: "🧾 관련 신고 Case"
- 자동 표시:
  - 연결된 모든 Business의 법인세 Case
  - 주업체 대표자(Person)의 종소세 Case
- 행: `[ABC법인 - 법인세 2024귀속] [결재대기] →` 열기 버튼
- **생성 버튼은 없음** (생성은 Person/Business 페이지에서만)

### 3.4 Filing Case 상세 페이지 (신규 — 핵심)

5단 구조 (위→아래):

#### A. 헤더
- 상단: `2024년귀속 [종소세] 신고검토표`
- owner 정보:
  - 종소세: `사람: 김대표 / 포함 사업체: ABC상사, XYZ카페, DEF디자인`
  - 법인세: `업체: 주식회사 ABC`
- 우측: 결재 상태 뱃지 + 작년 Case 링크 (있으면)

#### B. 비교표 — 작년 vs 올해

| 항목 | 2023귀속 | 2024귀속 | 증감액 | 증감률 |
|---|---|---|---|---|
| 수입금액 | | | | |
| 소득금액 | | | | |
| 산출세액 | | | | |
| 공제·감면 | | | | |
| 결정세액 | | | | |
| 가산세 | | | | |
| 기납부세액 | | | | |
| 납부할세액 | | | | |
| 실효세율 | | | | |

- 작년 Case가 없으면 작년 컬럼 비우고 "첫 도입 신고건" 표기.

#### C. 작년 리뷰 / 올해 리뷰 (좌우 2단)

| 작년 리뷰 (2023) | 올해 리뷰 (2024) |
|---|---|
| ○ 적용 공제감면 (목록) | ○ 적용 공제감면 (목록) |
| ○ 당시 특이사항·이슈 | ○ 직원 코멘트 |
| ○ 작년 결재 코멘트 | ○ 결재자 코멘트 (입력칸) |

→ "적용 공제감면"은 작년/올해 비교해서 신규/유지/삭제 자동 표시.

#### D. 입력 영역 (직원 작성용)

- **수기 입력 폼**
  - 신고 종류(종소세/법인세)에 따라 필드 자동 노출
  - 숫자 입력 칸 (천 단위 콤마 자동 포매팅)
  - 공제·감면 / 가산세는 행 추가/삭제 가능 (항목명 + 금액)
  - 실효세율은 자동 계산 (결정세액 ÷ 수입금액)
  - 임시 저장 지원 (작성 중 자동 저장)
- **수기 메모 입력**
  - 기존 메모 시스템 그대로, attach_to=Filing, category='신고 코멘트' default
  - 통합 메모로 자동 전파
- **상태 변경 버튼**: '작성중' → '결재대기' (직원이 작성 마치면 클릭)

> 💡 향후 확장: 자동 추출(신고서 PDF → Claude) 검증 완료 후 이 폼에 "PDF로 채우기" 버튼 추가 예정

#### E. 결재 영역 (사장님용) — 종이 기반

**결재의 본질은 종이.** 앱은 보조 도구.

사장님 동선:
1. 비교표 / 작년 리뷰 / 올해 리뷰 화면에서 검토
2. (선택) 결재자 코멘트 입력 — 다음 해 "작년 리뷰"에 검색 가능하게 하려는 용도. 빈칸 가능
3. **검토표 PDF 출력** 버튼 클릭 → A4 가로 인쇄
4. 종이에 손으로:
   - ☐ 필수 검토 체크박스 표시
   - 손글씨 코멘트 (필요 시)
   - **도장 4개 (담당자 / 사무장 / 세무사 / 대표)**
5. 앱에서 **"보관완료"** 버튼 클릭 + 결재일자 입력
6. 종이는 보관함에

#### F. 결재 상태 (3단계)

`작성중` → `결재대기` → `보관완료`

- **작성중**: 직원이 폼 작성 중. 직원만 편집 가능
- **결재대기**: 직원 완료. 사장님 검토·인쇄·결재 단계
- **보관완료**: 사장님이 PDF 출력하고 종이에 결재 도장 다 받음. 이후 read-only

## 4. 수기 입력 폼

직원이 신고 마치고 Filing Case 페이지에서 직접 입력. 모든 금액 필드는 천 단위 콤마 자동 포매팅 + 숫자만 허용.

### 4.1 종소세 입력 폼

**기본 금액 필드 (단일 입력)**
- 수입금액
- 소득금액
- 종합소득공제
- 과세표준
- 산출세액
- 결정세액
- 기납부세액
- 납부할세액

**반복 입력 (행 추가/삭제 가능)**
- 세액공제·감면 — `[항목명] [금액]` (예: 자녀세액공제 / 600,000)
- 가산세 — `[항목명] [금액]` (예: 무신고 / 50,000)

**자동 계산 필드 (입력 불가, 표시만)**
- 실효세율 = 결정세액 ÷ 수입금액 × 100

### 4.2 법인세 입력 폼

**기본 금액 필드**
- 매출액
- 결산서 당기순이익
- 각사업연도소득금액
- 과세표준
- 산출세액
- 결정세액
- 기납부세액 (중간예납 등)
- 납부할세액

**반복 입력**
- 공제·감면 — `[항목명] [금액]`
- 가산세 — `[항목명] [금액]`

**자동 계산**
- 실효세율 = 결정세액 ÷ 매출액 × 100

### 4.3 폼 동작
1. 직원이 Filing Case 생성 → 빈 폼 노출
2. 직원이 신고서 보면서 수기 입력
3. 작성 중 자동 저장 (5초마다 또는 필드 이탈 시)
4. 직원이 검토 후 코멘트 작성 → "결재대기" 버튼
5. 사장님 결재 → "결재완료"

### 4.4 향후 확장 (이번 범위 아님)
- 신고서 PDF 업로드 → Claude 자동 추출 → 폼 자동 채우기
- 단, 추출 정확도 검증 완료 후 도입

## 5. 비교 로직 (작년 Case 자동 참조)

### 5.1 자동 참조 규칙
- 새 Case 생성 시:
  - 같은 owner_type + owner_id + type, 직전 귀속연도 Case 자동 검색
  - 있으면 비교표에 작년 데이터 자동 채움
  - 없으면 "첫 도입" 표기

### 5.2 비교 항목
- 자동 필드는 그대로 좌우 비교 + 증감액/증감률 자동 계산
- 적용공제감면은 양쪽 항목 매칭:
  - 작년 O / 올해 O → "유지"
  - 작년 X / 올해 O → "신규"
  - 작년 O / 올해 X → "삭제"
- 실효세율 변동 강조 (절대값 0.5%p 이상 변동 시 색상 표시)

## 6. 메모 통합 — 어디서 뭐가 보이나

| 페이지 | 보여줄 메모 (인덱스 매치) |
|---|---|
| Person | related_persons에 본인 포함된 모든 메모 |
| Business | related_businesses에 본인 포함된 모든 메모 |
| ChatRoom | related_chatrooms에 본인 포함된 모든 메모 |
| Filing | related_filings에 본인 포함된 모든 메모 (집중뷰) |

각 메모에 출처 뱃지 표시: `[ABC법인]` `[종소세 2024]` `[김대표]` 등 → 클릭 시 해당 엔티티 페이지로 이동.

## 7. 검토표 PDF 와이어프레임 (A4 가로, 1~3페이지 가변)

기본은 1페이지. 코멘트가 길어지면 자동으로 2~3페이지까지 확장.

### 7.1 페이지 분배 원칙

**Page 1 (항상 출력)** — 결재 보고 손으로 채우는 핵심 양식
- 헤더 (로고 + 제목 + 도장 4개 영역)
- 기본정보 (사람·사업체)
- 비교표 (작년 vs 올해)
- 작년/올해 리뷰 — **공제감면 목록 + 짧은 코멘트(약 5줄 이내)**
- 필수 검토 체크박스
- 결재 도장 영역

**Page 2~3 (있을 때만 출력)** — 별첨: 상세 리뷰 메모
- 작년 리뷰 코멘트 전체
- 올해 리뷰 코멘트 전체
- 결재자 코멘트
- 직원 추가 메모 / 이슈 / 특이사항

→ Page 1의 리뷰 영역에는 **요약만** 표시하고, 본문이 길면 "▷ 별첨 참조 (P.2)" 표시 후 Page 2로 흘림.

### 7.2 페이지 분할 방지 (필수)

**박스·표·단락이 페이지 경계에서 절대 끊어지지 않게.** docx로 생성 시 다음 옵션 적용:

- 표는 행 단위 `cantSplit` (한 행이 두 페이지에 걸치면 안 됨)
- 단락은 `keepLines` (한 단락이 두 페이지에 걸치면 안 됨)
- 헤더·제목은 `keepNext` (제목과 본문이 분리되면 안 됨)
- 도장 4개 박스는 한 덩어리로 묶음
- SECTION 헤더와 그 아래 첫 박스는 항상 같은 페이지에

### 7.3 Page 1 와이어프레임

```
┌────────────────────────────────────────────────────────────────────────┐
│ [로고]  2024년귀속 [종소세] 신고검토표         [담당][사무장][세무사][대표]│
├────────────────────────────────────────────────────────────────────────┤
```

## 8. 첫 해(2024귀속) 운영 가이드

작년(2023귀속) Case 데이터가 없으므로:

1. **권장**: 직원이 작년 신고서 보고 retro 수기 입력
   - 자동 필드만 채워넣음 (작년 매출/소득/세액/공제감면)
   - 코멘트는 비워두거나 기억나는 부분만 입력
   - 1년치만 노가다하면 내년부터는 자연스럽게 비교 가동
2. 또는 작년 데이터 없이 진행 → 비교표는 "첫 도입" 표기
3. 내년(2025귀속)부터는 자동으로 작년 Case 참조

## 11. 결정 사항 요약

| 항목 | 결정 |
|---|---|

| 종소세 위치 | Person 페이지 |
| 법인세 위치 | Business 페이지 |

| **결재 방식** | **종이 기반** (앱은 상태 추적 + PDF 생성). 도장 4개 (담당자/사무장/세무사/대표) |
| **결재 상태** | 작성중 → 결재대기 → 보관완료 (3단계) |
